# Таксономія класів — ЧЕРНЕТКА (v2: tiers за дистанцією)

> **Статус:** Draft v2. Потребує підтвердження від команди + параметрів камери.
> Узгодити фінал ДО початку розмітки нових кадрів.
> Див. також: `docs/pixel_budget.md` (GSD-таблиці).

## Перехід v1 → v2

v1 була плоским списком 10 класів. v2 визнає, що **набір валідних класів залежить від режиму зйомки (висота × FOV)**. Один плаский список ламається бо половина класів фізично невидима на верхніх висотах.

## Tier 1 — велика техніка (виживає на 800 м)

З оптичним зумом — точно. Широким об'єктивом — лише до ~400 м.

| ID | Code | UA назва | Приклади |
|---:|---|---|---|
| 0 | `tank` | Танк | T-72, T-80, T-90, T-62 |
| 1 | `ifv_apc` | ББМ / БТР | BMP-1/2/3, BTR-80/82, MT-LB, BMD |
| 2 | `artillery` | Артилерія (towed + SP, без розділення на v1) | D-30, 2A65, 2S19, 2S3, 2S1 |
| 3 | `air_defense` | ППО | TOR, BUK, Pantsir, S-300/400, ZU-23 |
| 4 | `mlrs` | РСЗВ | Grad, Uragan, Smerch, TOS-1/2 |
| 5 | `truck_logistics` | Військовий вантажний транспорт | KamAZ-5350, Ural-4320 |
| 6 | `radar_ew` | Радари / РЕБ | Kasta, Krasukha, Murmansk-BN |
| 7 | `destroyed_vehicle` | Спалена / знищена техніка | (раніше hard negative, тепер окремий клас) |

## Tier 2 — легка / дрібна техніка (тільки низькі висоти або зум)

Широким об'єктивом вище ~300 м не мітимо — це порожні ground-truth, які лише шумлять.

| ID | Code | UA назва | Приклади |
|---:|---|---|---|
| 8 | `light_vehicle` | Цивільні легковики / Tigr-like SUV | без розрізнення військовий/цивільний на dist |
| 9 | `motorcycle` | Мотоцикли, квадроцикли | |
| 10 | `infantry` | Піхота / скупчення людей | ≥3 у військовому контексті |

## Зміни проти v1

- `artillery_towed` + `artillery_sp` → один клас `artillery` (різниця нерозбірлива на дальніх px-бюджетах).
- `destroyed_vehicle` піднятий з hard negative у клас — за вимогами епіка треба явно розрізняти спалену техніку як **не-ціль**, що означає її треба детектувати.
- `uav` прибрано — наземні Shahed launcher → це окремий збір, літаючі БПЛА в межах 300–800 м це окрема задача (air-to-air).
- Додано `mlrs` як окремий (раніше малось через `artillery_*`).

## Hard negatives (без боксів)

- **Цивільний транспорт у місті/дорозі** (поза тактичним контекстом).
- **Порожній ландшафт** (посадка, ліс, поле, місто).
- **Поодинокі цивільні людини** (1-2).

## Дві конфігурації моделі (можливе фінальне рішення)

1. **«Висотна/далека»** — Tier 1 тільки (8 класів). Для режиму огляду з 800 м, fixed-wide або zoom-out.
2. **«Низька/ближня»** — Tier 1 + Tier 2 (11 класів). Для режиму наведення на конкретний район з ≤300 м або zoom-in.

Це не обов'язково дві окремі моделі — може бути одна модель, але training дані стратифіковані так, щоб Tier 2 мав достатнє представлення тільки в низьких висотах.

## Відкриті питання (blocker)

1. **Параметри камери** — без них Tier-розподіл орієнтовний. Див. `docs/pixel_budget.md`.
2. **`destroyed_vehicle` як клас vs hard negative** — підтвердити з командою (раніше було hard negative за гайдлайном).
3. **`artillery` плоский чи розділений** — на 800 м не розрізнити, на 300 м — можна. Рішення залежить від цільового режиму.
4. **800 м — для всіх Tier 1 чи саме для ППО?** — від цього залежить чи всі великі класи треба стратифікувати до 800.
5. **OBB на v1 чи v2?** — для oblique view dalnіх цілей axis-aligned bbox тягне багато фону. Рішення впливає на формат розмітки (polygon-like 4-точкові OBB vs стандартні YOLO bbox).
